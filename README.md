# ⟳ leetrevive

> **Language / 语言：** English · [中文](README_CN.md)

A terminal-first, local-first LeetCode spaced-repetition assistant.

`leetrevive` turns your solved problems into a personal review bank. Each day it picks which problems to revisit — prioritising the ones you're weakest on and the ones coming due — so your knowledge actually sticks before interviews.

**No accounts. No scraping. No subscriptions.** Your data lives in a single SQLite file on your machine.

## Features

| | |
|---|---|
| 📥 **Auto-fill metadata** | `leetrevive add 146` — title, difficulty, and tags are filled from bundled data (3 892 problems, no internet needed) |
| 🧠 **Spaced repetition** | SM-2-inspired scheduler; review intervals grow automatically as you score higher |
| 💡 **One-sentence insights** | Every problem ships with a short algorithmic hint generated by Claude |
| 🌐 **Web UI** | `leetrevive serve` opens a browser dashboard: Today cards + sortable problem bank |
| 🔗 **Dual links** | Every problem links to both **LeetCode.com** and **力扣 (leetcode.cn)** |
| 🈳 **Bilingual** | Web UI supports EN / 中文 toggle; [full Chinese README here](README_CN.md) |
| 🗄️ **Local-first** | Single SQLite file; override path with `LEETREVIVE_DB_PATH` |

## Quick Start

### 1 — Install

```bash
# Python ≥ 3.10 required
git clone https://github.com/shuo956/lc_helper.git
cd lc_helper/.claude/worktrees/competent-shaw

python3 -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate

pip install --upgrade pip
pip install -e ".[web]"          # add [dev] for pytest
```

### 2 — Initialise database

```bash
leetrevive init
# ✓ Database initialised at ~/.local/share/leetrevive/leet.db
```

### 3 — Add problems

```bash
# By ID only — everything else is auto-filled
leetrevive add 1       # Two Sum (Easy)
leetrevive add 146     # LRU Cache (Medium)
leetrevive add 200     # Number of Islands (Medium)
leetrevive add 23      # Merge k Sorted Lists (Hard)

# Override any field manually
leetrevive add 999 "My Custom Problem" --difficulty medium
```

### 4 — See today's picks

```bash
leetrevive today
```

Attempt each problem, then record your result:

```bash
leetrevive done 1 --score 2
leetrevive done 146 --score 3 --minutes 12 --note "hashmap + doubly-linked list"
```

**Score guide:**

| Score | Meaning |
|------:|---------|
| 0 | Complete blank — couldn't start |
| 1 | Partial — needed hints or didn't finish |
| 2 | OK — solved with minor hesitation |
| 3 | Clean — solved fluently and within time |

### 5 — Launch the web UI

```bash
leetrevive serve
# → opens http://localhost:8080 automatically
```

## CLI Reference

| Command | Description |
|---------|-------------|
| `leetrevive init` | Create / migrate the local SQLite database |
| `leetrevive add <id> [title]` | Add a problem (title/difficulty auto-filled from bundled metadata) |
| `leetrevive today` | Show today's 3 scheduled review problems |
| `leetrevive done <id> --score N` | Record a review session |
| `leetrevive review <id>` | Show full problem details + full review history |
| `leetrevive bank` | List every problem in your bank |
| `leetrevive open <id>` | Open the LeetCode URL in your browser |
| `leetrevive stats` | Aggregate stats: streak, totals, difficulty breakdown |
| `leetrevive serve [--port N]` | Launch the web dashboard (default port 8080) |

### `add` flags

```bash
leetrevive add <id> [title]
  --difficulty  easy|medium|hard
  --pattern     TEXT    # algorithm pattern tag (e.g. two-pointers)
  --url         TEXT    # override LeetCode URL
  --source      TEXT    # label for where you found the problem
```

### `done` flags

```bash
leetrevive done <id>
  --score    0|1|2|3   (required)
  --minutes  INT       # time spent solving
  --note     TEXT      # free-form note
  --mistake  TEXT      # what went wrong / what to remember next time
```

## Web UI Guide

```bash
leetrevive serve            # port 8080
leetrevive serve --port 9000
```

### 📅 Today tab

Click **⚡ Generate Today's Problems** to load your 3 picks.

Each card shows:
- Problem number, title, difficulty badge, algorithm tags
- 💡 One-sentence insight (key algorithm point)
- Reason for pick (overdue / due today / reinforcement)
- **LeetCode ↗** → leetcode.com
- **力扣 ↗** → leetcode.cn
- **Mark Done** → inline score panel (no page reload)

After submitting a score the card shows when this problem is due next.

### 📚 My Bank tab

**Add a problem:**
1. Type a problem ID (e.g. `146`) in the search box
2. Title + difficulty preview appears automatically
3. Press **Enter** or click **Add**

**Problem table:**
- Click any column header to sort (ID / Title / Difficulty / Next Due / Reviews)
- Filter box searches across title, tags, difficulty
- Due-date colours: 🔴 overdue · 🟡 today · grey = future date
- **Done** button on each row opens an inline score panel

**Language toggle (EN | 中文)** — top-right corner, saved to localStorage.

## Spaced Repetition Algorithm

| Score | Next review interval |
|------:|----------------------|
| 0 | 1 day (hard reset) |
| 1 | 3 days |
| 2 | interval × 2.0 (caps at 30d after 5 consecutive OKs) |
| 3 | interval × 2.5 (caps at 60d after 5 consecutive Cleans) |

**Daily pick waterfall** (highest priority first):

1. **Overdue** — past due date
2. **Due today** — scheduled for today
3. **Long unreviewed** — added ≥ 14 days ago, never attempted
4. **Reinforcement** — random pick from weakly-scored problems

## Data Pipeline

### Rebuild bundled metadata

```bash
pip install requests
python scripts/fetch_meta.py
# → src/leetrevive/data/problems_meta.json  (~1.2 MB, 3892 problems)
```

### Regenerate one-sentence insights

```bash
export ANTHROPIC_API_KEY=sk-ant-...
pip install anthropic
python scripts/generate_insights.py          # generate all
python scripts/generate_insights.py --resume # continue from last batch
# → src/leetrevive/data/insights.json
```
